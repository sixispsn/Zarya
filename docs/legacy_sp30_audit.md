# Аудит legacy/sp30_calculator.html

`legacy/sp30_calculator.html` является исходным прототипом расчётного ядра Zarya, но не нормативом. Нормативным источником остаётся СП 30.13330.2020; legacy используется для восстановления сценариев, справочных данных и контрольных чисел.

## Что перенесено с сохранением алгоритма

| Блок legacy | Реализация Zarya | Результат проверки |
|---|---|---|
| Таблица А.2, 29 групп потребителей | `app/data/sp30_tables.py` | Перенесена |
| Таблица Б.2 и интерполяция alpha | `app/data/sp30_tables.py:get_alpha` | Перенесена; есть контроль точек и интерполяции |
| Секундные, часовые и суточные расходы | `app/calc/water_demand.py` | Алгоритм `calcBlock` перенесён; golden-сценарий «офис, 480 чел.» совпадает |
| Подбор водомеров | `app/calc/water_meters.py` | Типоразмеры, S и проверки а/б/в перенесены |
| Полив | `app/calc/irrigation.py` | Формулы и нормы перенесены |
| Дождевой сток с кровли | `app/calc/storm.py` | Формула, климатические параметры и периоды перенесены |
| ВПВ | `app/calc/fire_*` | Legacy-таблицы развиты до геометрии, гидравлики и проверки кольца |
| Насосные кривые | `app/data/pumps.py` | Архивная тестовая база перенесена |

## Исправленные или изолированные недостатки legacy

1. В `calculate()` обращение к `log` выполняется до `const log = []` при включённом коэффициенте 0,6. В Python этот режим работает и покрыт тестом.
2. Legacy всегда принимал расход прибора с максимальным водоотведением `q0s = 1,6 л/с`. По п. 5.5 и формуле (5) СП 30 q0s выбирается по таблице А.1 для фактического состава приборов. В Zarya он стал явным входом `sewage_max_fixture_lps`.
3. Legacy строил кривую повысительного насоса со статической составляющей `Hстат = Hгар`. Это смешивает доступный напор источника с напором, который должен создавать насос. В Zarya используется `max(Hgeom + Hпр - Hгар, 0)` и динамические потери `kQ²`.
4. Значения `Hгар = 20 м` и `NPSHa = 8 м` в legacy были демонстрационными defaults. В проектном расчёте Hгар обязателен из ТУ, NPSHa проверяется только при наличии исходных данных. `Hпр = 20 м` сохранён как свободный напор перед диктующим прибором.
5. Автоматическое `Hgeom = (n - 1) * hэтажа + 1` допускается только как предварительная оценка. Для ПЗ Zarya требует отметки или явно рассчитанный Hgeom.
6. Четыре насосные кривые Grundfos являются архивной тестовой базой, а не достаточным каталогом для проектного подбора.

## Трассируемость водоснабжения и водоотведения

- Водоснабжение: группы потребителей -> таблица А.2 -> вероятностный расчёт -> расходы В1/Т3 -> водомер -> потери водомера -> Hтр.
- Напор: `Hтр = Hgeom + sum(Hil) + Hпр + sum(Hвод) + Hтепл + Hlввод`; напор насоса определяется сравнением Hтр и Hгар.
- Хозяйственно-бытовые стоки: `qs = qtot + q0s`, где q0s является явным исходным параметром.
- Дождевые воды: отдельный расчёт К2 по площади кровли, стенам и климатическим параметрам.
- Полив включается в водопотребление, но не включается в хозяйственно-бытовые стоки.
