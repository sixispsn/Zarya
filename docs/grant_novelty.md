## 1. Суть проекта (короткая формулировка, ~500 знаков)

«Заря» — веб-сервис автоматизированного проектирования внутреннего
противопожарного водопровода (ВПВ) зданий. По инженерному описанию объекта
(тип здания, этажность, помещения, трассировка сети) сервис выполняет полный
нормативный и гидравлический расчёт по СП 10.13130.2020 и автоматически
формирует комплект проектной документации раздела ИОС2: пояснительную записку,
гидравлический расчёт, спецификацию, принципиальную схему и проверку
живучести сети — пять документов в PDF с оформлением по ГОСТ 21.101.
Комплект предназначен для проверки и выпуска ответственным проектировщиком;
сервис не подменяет его подпись и не заявляется как автономная экспертиза.

---

## 2. Проблема

Проектирование ВПВ — рутинная, но ответственная задача каждого проекта
здания выше нормативных порогов. Сегодня проектировщик вручную:

- определяет число расчётных струй по табл. 7.1/7.2 СП 10.13130.2020;
- расставляет пожарные краны с проверкой покрытия (радиус компактной струи,
  длина рукава, правило разных стояков для длинных коридоров — п. 6.2.2);
- считает гидравлику сети (для кольцевых магистралей — итерационная увязка,
  которую вручную почти никто не делает честно);
- подбирает диаметры по предельным скоростям и насосное оборудование;
- оформляет комплект документации (дни работы).

Существующие инструменты — электронные таблицы и автономные калькуляторы —
дают отдельные цифры, но не документацию; расчёт кольцевых сетей и поверочные
аварийные режимы в массовых инструментах отсутствуют.

## 3. Техническая новизна (что реализовано и работает)

1. **Полный цикл «описание → комплект документов» за один запуск.**
   Прозрачный конвейер: нормативный анализ → расстановка ПК → гидравлика →
   аудит диаметров → пять PDF-документов. Время цикла ~15 секунд.

2. **Нормативный движок, кодирующий букву СП.** Таблицы 7.1 и 7.2
   СП 10.13130.2020 реализованы полностью (все строки, включая сноску о
   приоритете высоты над этажностью); правило п. 6.2.2 (разные стояки для
   коридоров длиной более 10 м); минимальные радиусы п. 7.15. Каждый результат
   несёт ссылку на конкретный пункт нормы — прослеживаемость для экспертизы.

3. **Гидравлический решатель кольцевых сетей.** Увязка кольцевой магистрали
   методом Хантера-Кросса; автоматическое распознавание топологии (дерево /
   кольцо); расчёт сетей с двумя вводами (двухконтурная увязка с учётом
   разности напоров вводов и обратных клапанов).

4. **Автоматическая проверка живучести** — моделирование отключения явно
   размеченных ремонтных секций между задвижками и отказа каждого ввода с
   пересчётом сети. Требуемая точка худшего режима выводится отдельно, но
   работоспособность за счёт насоса подтверждается только после проверки
   фактической рабочей точки оборудования.

5. **Архитектура ввода, развязанная от модели.** Единый слой намерения
   (Request DTO → Builder) принимает данные из веб-формы и YAML-файлов;
   дорожная карта — импорт из IFC/BIM и табличных форматов без изменения ядра.

6. **Инженерная достоверность.** Более 500 автоматических тестов: физические
   инварианты (нулевая невязка по замкнутому контуру, узловые балансы,
   распределение расхода по сопротивлениям плеч), нормативные случаи по
   строкам таблиц СП, геометрия документов (поля, рамки, штампы по ГОСТ).

## 4. Что получает пользователь (продукт)

- Веб-форма: описание объекта в терминах проектировщика, без формул.
- Кнопка «Рассчитать» → через ~15 секунд комплект из пяти PDF:
  1) Пояснительная записка (текст с нормативными ссылками);
  2) Гидравлический расчёт (лист с таблицей участков, А4, рамка+штамп);
  3) Спецификация оборудования и материалов;
  4) Принципиальная схема (А1);
  5) Проверка живучести кольцевой сети (поверочные аварийные режимы).
- Сохранение проектов, повторное открытие и пересчёт; проект — текстовый
  файл (YAML), пригодный для версионирования и обмена.

## 5. Статус разработки

Работающий прототип: ядро расчётов, генерация всех пяти документов,
веб-интерфейс, хранение проектов. Прототип прошёл обкатку на реалистичном
объекте (17-этажный жилой дом, двухвводная кольцевая сеть) с вычиткой
выходной документации.
