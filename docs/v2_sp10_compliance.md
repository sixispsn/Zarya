# Обязательный аудит В2 по СП 10.13130.2020

Нормативный источник предоставлен пользователем: `SP10.13130.2020.pdf`, утверждён приказом МЧС России от 27.07.2020 № 559. В2 считается соответствующим только после прохождения двух независимых проверок:

1. Числовой parity с блоком ВПВ в `legacy/sp30_calculator.html`.
2. Нормативная применимость и полнота по СП 10.13130.2020.

## Чек-лист

- область применения и исключения СП 10;
- выбор варианта ВПВ и допустимость объединения с другими системами - раздел 6;
- число одновременно действующих ПК-с и минимальный расход диктующего ПК-с - таблицы 7.1 и 7.2;
- расход, давление у диктующего клапана, DN, диаметр насадка и длина рукава - таблица 7.3;
- требуемая высота/радиус компактной части струи - пункт 7.15;
- размещение ПК, высота установки клапана и проверка покрытия - раздел 6.2;
- схема питания, кольцевание, секционирование, запорная арматура и сменность воды - раздел 6.1;
- необходимость, резервирование, пуск, электроснабжение и контроль пожарных насосов - раздел 12;
- материалы, монтаж, уклоны, крепления, испытания и маркировка трубопроводов - раздел 14;
- давление свыше 0,45 МПа у ПК и необходимость диафрагм/регуляторов - пункт 7.5;
- отражение всех принятых исходных данных, расчётного сценария и результатов в ПЗ и графической части стадии П.

Значение не считается подтверждённым только потому, что оно присутствует в legacy. Сначала проверяется, что выбранная строка таблицы и условие СП 10 применимы к конкретному объекту. Запрещено выбирать ближайшее значение или экстраполировать таблицу без прямого нормативного основания.

## Результат сверки расчётного ядра

- Таблица 7.3 содержит 108 допустимых сочетаний DN, насадка, длины рукава и высоты компактной части струи. Для каждой строки действует автоматический parity-тест между Python и canonical legacy.
- В исходном legacy была пропущена строка DN 65 / насадок 13 мм / струя 20 м. По таблице 7.3 восстановлены значения: расход 4,0 л/с; давление 0,464 / 0,467 / 0,470 МПа для рукавов 10 / 15 / 20 м.
- Значения 2,5 л/с в таблицах 7.1 и 7.2 являются нормативным минимумом. Значение 2,6 л/с для типового выбранного ПК берётся из таблицы 7.3. Поэтому валидный типовой расчёт legacy `n × 2,6` сохранён, но при отсутствии сочетания оборудования показывается только минимум `n × 2,5` без вымышленного давления.
- Для производственных и складских зданий высота обязательна: таблица 7.2 применима при высоте до 50 м включительно. При высоте свыше 50 м и объёме свыше 150 тыс. м³ применяется пункт 7.13: не менее четырёх ПК-с по 5 л/с каждый.
- Выбранная высота компактной части струи проверяется по пункту 7.15: не менее 6 м; для жилых зданий выше 50 м — 8 м; для общественных, производственных и административных зданий выше 50 м — 16 м.
- При давлении у ПК более 0,45 МПа результат содержит обязательный признак установки диафрагмы или регулятора давления по пункту 7.5.

Расчёт гидравлических потерь сети, диктующего пути и подбора пожарного насоса выполняется только при наличии явного графа/аксонометрии В2. На стадии П без трасс и отметок в ПЗ фиксируются расчётные расходы, требуемое давление у ПК, нормативная схема и исходные данные для последующей гидравлики; фиктивная длина сети не принимается.

## Кольцо, вводы и аварийные режимы

- Однокольцевая сеть увязывается методом Хантера-Кросса для принятой legacy-модели потерь `h = A·Lэфф·Q²`. Узловые напоры восстанавливаются со знаком расхода; обратное плечо не суммируется по модулю.
- Ремонтный отказ рассчитывается по явно заданной секции между запорными устройствами (`repair_section_id`). Без разметки выполняется только предварительная проверка отдельных сегментов с предупреждением; выпускной вердикт не формируется молча.
- Для каждой секции контролируются ограничения пунктов 6.1.12 и 13.1: не более полукольца и, при заданной принадлежности стояков, не более пяти стояков/опусков.
- Для двух вводов штатная увязка использует полные пьезометрические напоры с учётом отметок. Дополнительно проверяется отказ каждого ввода. При отключённой ремонтной секции применяется консервативная проверка полного расхода от каждого ввода по отдельности.
- `needs_pump=True` означает только дефицит напора. Живучесть за счёт насоса подтверждается исключительно переданной проверенной рабочей точкой, обеспечивающей расход и напор худшего режима; требование установки насоса само по себе подтверждением не является.
