# Матрица соответствия расчётов legacy SP Calculator

Канонический источник: `legacy/sp30_calculator.html`. Статус «parity» означает совпадение исходных таблиц, формул, ветвлений, округления и контрольных числовых результатов.

| Блок | Legacy-источник | Реализация Python | Проверка | Статус |
|---|---|---|---|---|
| Расходы общий/ХВС/ГВС | `TBL`, `B2`, `getAlpha()`, `calcBlock()` | `app/data/sp30_tables.py`, `app/calc/water_demand.py` | `test_water_demand.py`, `test_legacy_sp30_source_parity.py` | parity |
| Тепловые нагрузки ГВС | `Q_max`, `Q_avg` в `calculate()` | `app/calc/water_demand.py` | `test_water_demand.py` | parity |
| Напор повысительного насоса | `calcPumpHead()` | `app/calc/pumps.py`, мост `calc_required_head()` | `test_pumps.py`, `test_rules.py` | parity: Hтр раскладывается в ПЗ, а в подбор передаётся тот же агрегат `Hgeom + ΣHl + Hпр - Hгар` |
| База и кривые насосов | `PUMP_CURVES` | `app/data/pumps.py` | прямое сравнение каждой характеристики и точки Q-H с HTML | parity |
| Рабочая точка и скоринг | `interpH()`, `findWorkingPoint()`, `buildEffectiveCurve()`, скоринг `calcPumpHead()` | `app/calc/pumps.py` | golden для одиночного, параллельного и последовательного режимов | parity |
| Теплоизоляция ГВС/ХВС | `T4_SP61`, `RNL_TABLE`, `TV4`, `calcGvs()`, `calcHvs()` | частично в модели/спецификации | требуется прямой перенос и golden-тесты | не завершено |
| В2 | блок СП 10 в HTML; нормативный приоритет - СП 10.13130.2020 (приказ МЧС № 559) | `app/calc/fire_*` | требуется пофункциональная и числовая сверка legacy плюс отдельный нормативный аудит по СП 10 | не завершено |
| Детальная гидравлика участков В1 | в данном HTML отсутствует; `ΣHl` вводится вручную | `app/calc/v1_hydraulics.py` | parity с данным HTML невозможен | отдельное расширение, не считать переносом legacy |

## Контрольный golden подбора В1

Для `Hgeom=25 м`, `ΣHl=5 м`, `Hпр=20 м`, `Hгар=20 м`, `Q=5 м³/ч`, одного повысительного насоса и `NPSHa=8 м` функции legacy дают:

- `Hp = 30 м`;
- `Hстат = 20 м`;
- `k = 0,4 м/(м³/ч)²`;
- принятый насос Grundfos CR 10-4;
- рабочая точка `Q=5,94 м³/ч`, `H=34,1 м`;
- итоговый score `160`.

Python выдаёт те же значения. Эти числа закреплены тестом и не должны меняться без явного решения пользователя об изменении эталона.
