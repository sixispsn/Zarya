# Zarya Demo Release 0.1 — границы заявления

## Что демонстрируется

- воспроизводимый путь `YAML/веб-форма -> расчёт -> пять PDF`;
- нормативный расчёт В2 по СП 10.13130.2020;
- гидравлика кольцевой сети и явно размеченных ремонтных секций;
- подбор пожарной насосной установки по рабочей точке основного режима;
- расходы В1/Т3 и требуемый напор по исходным данным стадии П;
- укрупнённые длины труб В1/Т3-Т4 по площади, точные длины сети В2 и
  теплоизоляция по алгоритму legacy SP calculator;
- оформление ПЗ, гидравлического расчёта, спецификации, схемы и листа живучести.

## Что не заявляется

- автоматический выпуск документации без проверки проектировщиком;
- замена исходных ТУ, планов АР и реальной трассировки;
- окончательный выбор оборудования по единственной архивной Q-H кривой;
- детальная циркуляционная гидравлика Т3/Т4 без аксонометрии стадии Р;
- готовность облачной коммерческой эксплуатации и многопользовательского режима.

## Воспроизведение

```bash
OSIFONT_PATH=/System/Library/Fonts/Supplemental/Arial.ttf \
  venv/bin/python scripts/build_demo_release.py
```

Выход: `demo_backup/`. Сборка завершается ошибкой, если сформировано не пять PDF
или текст документов содержит незаполненные поля `⟦...⟧`.
