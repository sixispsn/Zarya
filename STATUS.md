## ТЕКУЩАЯ СЕССИЯ: библиотека УГО для принципиальной схемы

### Главный вывод (важно!)
Принципиальная схема для нашего объекта делается КАК В РЕАЛЬНОМ ПРОЕКТЕ Антона
(файл том_В-14.pdf — «Конференц-зал», стадия П), а НЕ как эталон 27-этажки из
ГОСТ 21.619 Прил.Б. Ключевое:
- схема ЧЁРНО-БЕЛАЯ (не цветная!);
- системы различаются БУКВЕННЫМИ метками вдоль труб (В1/Т3/Т4/В2), не цветом;
- стояки — вертикали ШТРИХПУНКТИРОМ с повторяющимися буквами системы;
- разрез здания НЕ нужен (был просто фоном в примере).
Прошлые попытки (цветная развёртка, функциональная блок-схема, копия эталона Б)
ОТВЕРГНУТЫ. Идём через поэлементную сборку УГО → потом схема.

### Метод работы по УГО (работает!)
Антон описывает символ словами или шлёт вырезку DWG → Claude рисует ОДИН символ
в SVG → cairosvg в PNG → сверка → правки до принятия. Английское текстовое
описание символа сработало особенно хорошо.

### СЕМЕЙСТВО «БАНТИК» (база всей арматуры)
Два равнобедренных треугольника, основания снаружи (вертикальные), вершины
встречаются в общем центре = bowtie/песочные часы. Отличия по центр. элементу:
- большой ПУСТОЙ кружок         → запорный вентиль
- вертикальный отрезок (длиннее)→ задвижка
- один треугольник ЧЁРНЫЙ        → обратный клапан (поток белый→чёрный)
- шток вверх + «шляпа гриба»(полукруг замкнут прямой=мембрана) + импульсная линия
  (вверх→коротко вправо→вниз) с крупной стрелкой к выходу → редуктор давления
- шток + ПУСТОЙ кружок на конце  → балансировочный клапан

### ПРИНЯТО И СОБРАНО в app/pz/ugo.py (10 символов):
Файл ugo.py готов (см. /mnt/user-data/outputs/ugo.py), каждая функция = SVG-строка,
чёрно-белая. Все приняты Антоном:
1. washbasin(x,y,s=34,side) — УМЫВАЛЬНИК: ромб + вертикаль внутри + некрупный
   закрашенный круг-сифон (r=0.32*s) + НОСИК вниз-вбок (side right/left, зависит
   от места установки) + толстый отвод вниз.
2. МОЙКА = washbasin (то же УГО, отличается только подписью).
3. shower(x,y,s=34,side) — ДУШ: умывальник + шланг РОВНЫМ полукругом (R=1.35*s)
   над прибором + лейка-треугольник вершиной ВВЕРХ (основание внизу) на конце дуги.
4. toilet(cx,top_y) — УНИТАЗ: вытянутый прямоугольник-бачок (150×75) + отвод ближе
   к КРАЮ (px=cx+bw*0.32) + вентиль-бабочка с малым кругом на отводе.
5. valve_h / valve_v — ЗАПОРНЫЙ ВЕНТИЛЬ: бантик + большой пустой кружок (dot=12).
6. gate_h / gate_v — ЗАДВИЖКА: бантик + отрезок в центре (bar=15, поперёк трубы).
7. check_h / check_v — ОБРАТНЫЙ КЛАПАН: один треугольник чёрный, параметр flow
   (right/left/down/up), поток от белого к чёрному.
8. reducer_h — РЕДУКТОР ДАВЛЕНИЯ: бантик + шток + шляпа-мембрана + импульс+стрелка.
9. balance_h / balance_v — БАЛАНСИРОВОЧНЫЙ КЛАПАН: бантик + шток + пустой кружок.
10. air_valve(x,y) — ВОЗДУШНЫЙ КЛАПАН: короткая вертикаль + треугольник вершиной вверх.

Общие параметры: BLK="#000", tri=26 (полуразмер бантика), w=2.4 (толщина).

### ОСТАЛОСЬ НАРИСОВАТЬ (ждут описания Антона):
- fnet — фильтр сетчатый (на узле учёта)
- meter — счётчик воды (узел учёта)
- pump — насос (насосная станция)
- pk — пожарный кран ПК (шкаф)
- riser — СТОЯК: вертикаль штрихпунктиром + буквенные метки В1/Т3/Т4 вдоль линии
  (нужен интервал меток и формат подписи Ø снизу)
- (опц.) ввод/врезка в сеть — стрелка/обозначение

### ПОТОМ — сборка схемы
Из ugo.py собрать app/pz/scheme.py: пресет «ЖИЛЬЁ» (унитаз+умывальник+душ+мойка)
× N стояков × M этажей (вопр.2=а: один типовой узел размножается; вопр.3: по
умолчанию однотипный по этажам, дифференциация позже). Стояки штрихпунктиром с
буквами сквозь этажи, отметки слева, низ: ввод→узел учёта→насос→магистраль,
циркуляция Т4, ПК. ВСЁ ЧЁРНО-БЕЛОЕ. Топологию приборов задаёт инженер (не
выдумывать) — пресеты приборов из библиотеки.

## Правила (критично)
- Источник истины для расчётов В1, В2, Т3 и Т4 — `legacy/sp30_calculator.html`:
  расходы, потери/напоры, теплоизоляция, водомеры и насосы переносятся один в один
  и подтверждаются parity-тестами. Новые формулы и коэффициенты не изобретать.
- В2 дополнительно и безусловно проверять по СП 10.13130.2020 (приказ МЧС России
  от 27.07.2020 № 559): область применения, таблицы 7.1-7.3, требования к сети,
  насосной установке и трубопроводам. Расхождение legacy и СП 10 предъявлять Антону.
- Кодить строго из текста СП/ГОСТ, не по памяти.
- SVG f-строки: НЕ писать f'<tag>' f'{v:.1f}'.replace(".",",") — соседние f-строки
  склеиваются ДО replace и ломают координаты. Считать в переменную: 
  mark_txt=("{:+.1f}".format(v)).replace(".",",").
- cairosvg: pip install cairosvg --break-system-packages.
- Скачивание файлов — через браузер claude.ai; в Chrome жёсткий refresh Cmd+Shift+R
  (SVG кэшируется между запусками!).
- Кнопки ask_user_input_v0 НЕ срабатывают у Антона — спрашивать текстом.
- В терминал НЕ копировать строки с комментариями (# ...) — zsh ругается.
